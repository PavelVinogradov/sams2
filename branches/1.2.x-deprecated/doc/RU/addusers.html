Запуск системы в эксплуатацию:

Установите необходимое ПО:
mysql,
squid,
samba 2.x или samba 3.x,
apache,
согласно документации, установите
sams
и попробуйте загрузить графический интерфейс SAMS. Если интерфейс пользователя недоступен или работает с ошибками, прочитайте здесь.

Введите пароль администратора (пользователь 'admin', пароль - 'qwerty')

Настройте WEB интерфейс:

Настройки WEB интерфейса и собственно SAMS разнесены в разные ветви, так как WEB интерфейс может использоваться не только SAMS

В левом фрейме выберите WEB interface settings Далее в нижнем фрейме нажмите на конпку (WEB interface settings)

В открывшемся диалоге выберите:

  • Language: (Язык) Язык (кодировка) веб интерфейса
  • Allow the users access to your own statisatics (Разрешить доступ пользователей к своей статистике): - Позволяет пользователям авторизоваться в WEB интерфейсе SAMS для просмотра своей статистики:
    - on daily traffic (по трафику за день) - показывает объем скаченного пользователем трафика по дням
    - on URL visiting (по посещенным URL) - показывает пользователю список посещенных им URL
  • Icon set (Тема иконок) - тема иконок WEB-интерфейса
  • Show users tree (Показывать дерево пользователей) - пользователи могут увидеть всех пользователей, занесенных в SAMS
  • Show users as (Показывать пользователей как) - выбрать как будут показываться пользователи (login или фамилия и т.д.)
  • Kilobyte size of your ISP (byte) (Размер Килобайта, учитываемого вашим провайдером (byte))
  • Megabyte size of your ISP (byte) (Размер Мегабайта, учитываемого вашим провайдером (byte) )
  • Create diagrams (Показывать графики в отчетах ) - в некотороых отчетах рисуются графики
  • Create PDF reports:(Создавать PDF отчеты)

    Настройте SAMS:

    В левом фрейме выберите SAMS -> "Администрирование SAMS" Далее в нижнем фрейме нажмите на конпку (Настройки SAMS)

    Смена пароля администратора:

    Администрирование SAMS -> Сменить пароль пользователя

    Создайте шаблоны пользователей.

    Создайте шаблоны пользователей. Шаблоны включают в себя списки запрета доступа, перенаправления запроса и скорости канала пользователей.

    Создайте группы пользователей.

    Создайте группы пользователей. Группы не несут никакой смысловой нагрузки и не привязываются, например к группам в домене, они созданы только для удобства работы с пользователями в интерфейсе SAMS.

    Зарегистрируйте пользователей в системе:

    Пункт "Пользователи".

  • Если у вас используется NTLM или AD аутентификация (в домене Windows), вы можете вводить пользователей из формы регистрации по-одному, или ввести сразу группу пользователей из контроллера домена:
    Пользователи -> Загрузить всех пользователей домена
    Выберите пользователей из списка, определите их в группу и задайте им шаблон.

  • Если у вас используется доступ к SQUID по ip адресам, вы можете вводит пользователей из формы регистрации. Необходимо задать имя пользователя, ip адрес и пароль для доступа пользователей к их статистике

  • Если у вас используется NCSA аутентификация, вы можете вводить пользователей из формы регистрации. Необходимо задать имя пользователя и пароль для доступа пользователей к их статистике

    Перезагрузите SQUID

    Для внесения изменений в настройки SQUID перезапустите SQUID
    Squid -> Реконфигурирование SQUID
