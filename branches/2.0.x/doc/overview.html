1 Принцип работы и краткое описание модулей

На Рис. 1 представлена принципиальная схема работы SAMS с применением встроенного редиректора. В случае использования других редиректоров, доступность ресурса определяется правилами, которые заносятся в конфигурационный файл squid

Рисунок 1

Весь SAMS разбит на 3 основных модуля, каждый из которых может быть установлен на разных машинах, при этом через один web интерфейс можно управлять разными демонами на разных машинах.

sams2 Предназначен для чтения журнала access.log у squid, настройки конфигурационного файла squid, автоматической очистки счетчиков пользователей, автоматической блокировки пользователей, перенаправления или блокировки URL запросов, и т.п.
sams2-web Предназначен для управления через web интерфейс, ограничивая возможности на основе уровня привилегий аутентифицированного пользователя. Позволяет управлять настройками демонов, менять правила доступа в Интернет, строить графики, предоставлять статистику посещений, генерировать отчеты в виде pdf файлов, показывать информацию об операционной системе, на которой работают демоны и squid.
sams2-doc Различная документация как по установке, настройке, управлению, так и по принципам и правилам написания дополнительных подключаемых модулей. Здесь же содержится документация по исходным текстам программ.
